Collaborator onboarding

Read this before changing anything.

Start from your access.

Use Live Survey for a real run-through, Google Sheets for questionnaire wording and structure, Data Examples for file shapes, and Plot Lab for public feedback-plot tuning.

Keep the edit contract intact.

Stable questions keep the same item_id. English rows carry structure and scale_id. Scorable submissions must produce all 11 canonical ASC factor scores.

Protect context and text.

The live feedback controls only expose canonical buckets: cannabis, psilocybin, alcohol, MDMA, and low, medium, high. Raw text and Other-style context never belong in participant dropdowns.

Expect quality review metadata.

Submitted responses include broad timing and operational metadata used to assess whether a response looks like a reasonable completed submission. This guide keeps the exact review criteria high-level so those signals remain useful.

Access types

  • Live survey access: open the live questionnaire, submit a response, use the comparison controls, and inspect or share the current feedback view.
  • Google Sheets access: review or edit questionnaire content in the shared sheet, including wording, translations, options, and version tabs.
  • OSF access: download the latest public or private files, plus codebooks and changelogs.
  • Some collaborators have all three. Others only have Google Sheets and OSF access.

Access note

Live survey here means the public or deployed survey page, not code or repo access.

Common tasks

I need to test the live flow

Use the live survey page.

I need to change wording or translations

Google Sheets access is usually enough for the edit itself. Keep one tab per version and keep translation rows aligned to the English structural rows.

I need the latest files

OSF access is enough to get the current bundle. Use the data examples page alongside the bundle when you need field shapes, quality gates, and formatting spelled out.

I only have Sheets and OSF access

You can still edit the questionnaire and use the downloads. Ask someone with live survey access to confirm the running flow after a structural change.

Fastest Path

Use the site in this order

  1. 1
    Identify your access level.

    Decide whether you can open the running survey, edit the sheet, download from OSF, or some combination of those.

  2. 2
    Open the matching page.

    Use Live Survey for the running questionnaire, Google Sheets for questionnaire changes, and Data Examples when you need file structures, field formats, and export shapes.

  3. 3
    Open the running survey once after a sheet change if you want confirmation.

    That is enough to see whether the updated questionnaire was picked up. For structural or scoring changes, complete one full submit path and check that feedback renders.
