Google Sheets Access

Review and update the questionnaire in the shared sheet.

Use this page if your main access is the shared questionnaire sheet. You can review wording, translations, options, and version tabs here. Anything that must be checked in the running survey still needs app access.

What you can do in Google Sheets

  • Review and update wording in the current questionnaire version.
  • Add or adjust translations in the shared table.
  • Update answer options and context choices.
  • Create a new dated version tab when a change should be tracked separately.
  • Prepare a clear handoff for someone who will verify the live app.
App access still required

Ask an app user to verify anything that depends on the running questionnaire, the submit flow, the feedback page, or the live comparison controls.

Change types and who needs to verify them

Change type Can start in Google Sheets? Needs app-level verification? Typical next step
Wording or translation cleanup Yes Usually yes Edit the sheet, then ask an app user to confirm the live wording if the result matters in the running survey.
Required or active toggle Yes Yes Check the live navigation and completion flow in the app.
Substance or dose option changes Yes Yes Verify the live option set and the feedback-side comparison behavior in the app.
New scorable item or factor remapping Yes Yes Create or update the dated version, then use app access to verify the live questionnaire and feedback path.

Google Sheets workflow

  1. 1
    Open the current shared questionnaire tab.

    Confirm you are working in the intended version before changing anything.

  2. 2
    Make the content change.

    Update the wording, translation, options, or structural rows that need to change.

  3. 3
    Keep stable questions stable.

    Keep the same item_id when the question is still the same underlying question, and keep scorable rows mapped through the intended scale_id.

  4. 4
    Create a new dated tab when the version truly changes.

    Use a new dated version when the questionnaire definition should be tracked separately.

  5. 5
    Hand off live verification.

    Ask someone with app access to reload the running questionnaire and confirm the change in the app.

  6. 6
    Tell download users if files may change.

    If the change affects the meaning of later exports or codebooks, let OSF users know what to expect.

When to ask for app-level help

  • When the change must be checked in the live questionnaire UI.
  • When the submit flow or required-step behavior might change.
  • When the feedback chart or comparison controls need verification.
  • When someone needs to reload the running questionnaire after a version update.
