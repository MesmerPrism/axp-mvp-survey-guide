Google Sheets Access

Review and update the questionnaire in the shared sheet.

Use this page if your main access is the shared questionnaire sheet. The sheet is the editable questionnaire source. Valid edits there are what the running survey reads. Live survey access is still needed when someone must confirm the running survey behavior or refresh an already open survey session.

What you can do in Google Sheets

  • Review and update wording in the current questionnaire version.
  • Add or adjust translations in the shared table.
  • Update answer options and context choices.
  • Create a new dated version tab when a change should be tracked separately.
  • Prepare a clear handoff for someone who will verify the live survey page.
What live survey access is for

Live survey access here means being able to open the running web survey page, not code or repo access. It is for live verification, not for making a valid sheet edit count. New or reopened survey sessions read the sheet directly. Ask someone who can open the live survey only when the running questionnaire, submit flow, feedback page, or an already open session needs checking.

Change types and when a live survey check is helpful

Change type Can start in Google Sheets? Needs a live survey check? Typical next step
Wording or translation cleanup Yes Sometimes Edit the sheet. New survey loads pick it up. Ask someone who can open the live survey only if the wording needs to be checked in the running questionnaire.
Required or active toggle Yes Yes Edit the sheet, then use live survey access to confirm the navigation and completion flow.
Substance or dose option changes Yes Yes Edit the sheet, then use live survey access to verify the option set and the feedback-side comparison behavior.
New scorable item or factor remapping Yes Yes Create or update the dated version in the sheet, then use live survey access to verify the questionnaire and feedback path.

Google Sheets workflow

  1. 1
    Open the current shared questionnaire tab.

    Confirm you are working in the intended version before changing anything.

  2. 2
    Make the content change.

    Update the wording, translation, options, or structural rows that need to change.

  3. 3
    Keep stable questions stable.

    Keep the same item_id when the question is still the same underlying question, and keep scorable rows mapped through the intended scale_id.

  4. 4
    Let the sheet enforcer sync the tab.

    If the Google Sheets Apps Script is installed, it will auto-sort language blocks, keep the expected header shape, and apply the row formatting the questionnaire uses.

  5. 5
    Create a new dated tab when the version truly changes.

    Use a new dated version when the questionnaire definition should be tracked separately.

  6. 6
    Use live survey access only when a live check is needed.

    New or reopened survey sessions already read the updated sheet. Use the survey reload control only if someone needs to refresh an already open session or test a different sheet tab override.

  7. 7
    Tell download users if files may change.

    If the change affects the meaning of later exports or codebooks, let OSF users know what to expect.

What the Google Sheets enforcer does

  • If installed, the Apps Script runs on edit and is also available from the AXP Sheet menu as Sync questionnaire sheet.
  • It keeps the questionnaire tab in the expected header order and auto-sorts each language block by the English row order.
  • It colors and formats rows from the English structural metadata, including active state and scale_id.
  • It checks that explicit induction_method_N and dose_options_N header pairs are complete before syncing.
  • It helps keep the sheet structurally valid and readable, but it is not a separate publish step. The running survey still reads the questionnaire from the sheet itself.

When to ask for a live survey check

  • When the change must be checked in the live questionnaire UI.
  • When the submit flow or required-step behavior might change.
  • When the feedback chart or comparison controls need verification.
  • When someone already has the app open and needs that existing session refreshed after a sheet change.
