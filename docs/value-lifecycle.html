Values Through Stages

How values are computed and saved.

The important split happens after submission: one path keeps the current participant’s scores in memory for immediate feedback, another writes durable rows to storage, and a later export path widens those stored rows into public and private bundles.

Stage overview

Stage Main format Why it matters
Definition load One normalized questionnaire table The system computes a definition hash from the loaded table and then filters by language.
Runtime questionnaire state All languages plus selected-language view The running survey knows which instrument version and language were actually loaded.
Submission assembly In-memory response table All responses are gathered before scoring and persistence.
Scoring Per-factor score rows Numeric values are mapped to factors using the questionnaire’s factor mapping.
Persistence Submission, numeric, text, and score tables Raw collection is stored in normalized database tables.
Feedback Current self-scores plus optional peer comparison inputs The participant sees immediate results without waiting for export.
Export Public or private bundles Public output removes free text and anonymizes identities; private output retains full detail.

Storage model

  • submission stores one survey session row with version and timing metadata.
  • response_numeric stores numeric answers keyed by item_id.
  • response_text stores free text and context text when that storage path is enabled.
  • score stores computed factor values.
  • peer_plot_submission_cache stores one context row per submission for peer comparison filtering.
Important distinction

The raw database is the source of truth for collection. The export bundles are the source of truth for distribution.

What feedback uses

The immediate results view uses the current submission’s computed scores already in memory. Peer comparison values are loaded separately and only when the feedback step is active.

  • Self-scores do not require rereading the just-written submission rows.
  • Peer comparison starts from peer_plot_submission_cache and uses canonical induction and dose buckets rather than raw free text.
  • If too few peers exist in the selected bucket, the system omits peer dots rather than fabricating comparison data.
