App Access

Use the live app and verify the running experience.

This page is for collaborators who can open the AXP app itself. Use app access whenever the task depends on the live questionnaire, a real submit flow, the feedback screen, or the running comparison controls.

Google Sheets workflow Data examples Diagrams

What app access lets you do

  • Open the current live questionnaire.
  • Choose a language and walk the actual running flow.
  • Submit a response and reach the feedback view.
  • Check how wording, options, and required behavior appear in the app.
  • Use the feedback comparison controls after submission.
  • Reload the running questionnaire when a sheet-backed version needs to be rechecked.
App-only work

If a collaborator cannot open the live app, they cannot confirm the real questionnaire flow, the final submit path, or the feedback screen. Those checks stay with app users.

Live app workflow

  1. 1
    Open the app URL you were given.

    Start in the running survey rather than a local or static copy.

  2. 2
    Choose your language.

    Check the live wording in the language that matters for the current task.

  3. 3
    Complete one experience path.

    Move through the questionnaire as a real user would.

  4. 4
    Submit at the final reveal step.

    Do not stop before submit if the task depends on feedback or downstream behavior.

  5. 5
    Review the feedback screen.

    Check the chart, comparison controls, and any other live result behavior that matters for the task.

  6. 6
    Report the result back to other collaborators.

    Confirm what worked, what still needs adjustment, and whether another reload or sheet edit is required.

Common app-only checks

Task Why app access is required What to confirm
Live wording and translation check The running questionnaire is the real user experience. The text, ordering, and language selection look correct in the live app.
Required-step and navigation check Only the app shows the actual live step flow and validation behavior. The user can move through the intended path and cannot skip required inputs accidentally.
Submit and feedback check The feedback view only appears after a live submit flow. The chart loads, the response completes, and the result view behaves as expected.
Comparison control check The comparison controls live on the feedback screen. The controls appear, use recognized comparison buckets, and respond correctly after submit.
Questionnaire reload after a sheet change The reload action refreshes an already open app session or lets an app user test a different sheet tab override. The already open app session shows the updated questionnaire, or the requested override tab is loaded for checking.

What app users hand back to other collaborators

  • Whether the live wording and ordering matched the sheet change.
  • Whether the flow completed and submitted successfully.
  • Whether the feedback screen behaved correctly after submit.
  • Whether another sheet edit or another reload is needed.
