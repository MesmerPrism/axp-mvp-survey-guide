OSF Downloads

Use the available files without app access.

This page is for collaborators who have OSF download access and may or may not also have Google Sheets access. Use OSF when you need current export files, factor-score files, codebooks, or changelogs.

Which files to open Download workflow When app access is still required

What you can do with OSF access

Download the latest bundles.

Use OSF to get the current public or private files you have permission to open.

Use codebooks and changelogs.

Open the supporting files to interpret columns and confirm when the bundle was refreshed.

Work without app access.

You do not need the live app just to retrieve files or read supporting documentation.

Which files to open

File type Use it for App access required?
Public latest export The current public wide dataset. No
Public factor-score export The current public long factor-score file. No
Private full export The restricted wide dataset for users who have private-file access. No
Codebook Column meanings and field descriptions. No
Changelog When the bundle changed and what the refresh contained. No

Typical OSF download workflow

  1. 1
    Open the OSF location you were given.

    Use the public or private area that matches your permission level.

  2. 2
    Download the latest file set you need.

    Choose the wide export, factor-score file, codebook, or changelog based on the task in front of you.

  3. 3
    Use the codebook to interpret columns.

    Do not guess field meanings when the codebook is already available with the bundle.

  4. 4
    Use the changelog to confirm freshness.

    Check when the bundle was last refreshed before treating it as current.

  5. 5
    Hand off live checks if needed.

    If the task turns into a runtime verification question, pass it to someone with app access.

When OSF access is not enough

  • When you need to test the live questionnaire itself.
  • When you need to submit a response and reach the feedback screen.
  • When you need to verify how a Google Sheets change appears in the running app.
  • When you need to confirm live comparison controls or feedback behavior.
