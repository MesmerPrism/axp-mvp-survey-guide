Public-Safe Mock Files

Example files for every lifecycle stage.

These CSV and JSON snippets are synthetic and simplified, but they use the same column names and stage boundaries that the app uses. The lifecycle diagram links here so a newcomer can move from a Mermaid box straight into a concrete file shape.

Definition In-memory rows MariaDB tables Peer plotting Distribution Open file viewer

How to read these examples

Datatype labels are explicit.

Each example calls out whether the shape is a CSV source file, an in-memory R data.frame, a MariaDB table export, or a wide distribution CSV.

The values are mock.

No participant data was copied into this repo. These files exist only to explain shape, naming, and the public/private split.

The stages are intentional.

The same answer appears in different shapes as it moves from definition, to runtime, to persistence, to peer plotting, to export.

1. Definition and runtime

The questionnaire starts as a stacked multi-language table. After loading, the runtime keeps both the full multilingual table and a selected-language view, plus metadata such as instrument version and definition hash.

4. Peer plotting context and plotted values

        
The peer plot does not draw raw text rows directly. It first uses cached canonical induction/dose buckets to identify eligible submissions, then queries matching ASC factor score rows, then renders those rows as peer rings and a peer median overlay.

        

          

            
mock-data/peer_plot_submission_cache.csv

            
The one-row-per-submission cache that keeps raw and canonical context plus completeness flags.

            
Datatype: MariaDB table excerpt shown as CSV. Shape: one row per submission with raw plus canonical induction/dose values.

5. Distribution outputs

        
The distribution stage widens the normalized rows into publication-ready CSV files. Public output anonymizes identity and drops free text. Private output keeps the raw submission id, timestamp, and text columns.

        

          

            
mock-data/public_export.csv and mock-data/public_factor_scores.csv

            
The public wide bundle plus the public factor-only long export. These are the publishable shapes.
